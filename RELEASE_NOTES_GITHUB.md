FSOMA Transformation Brief Skill, v0.9.0 Public Preview

Public preview on GitHub. Not yet listed on the Claude marketplace.

What this skill does

Turns client operating model data, workflows, tech stack, RACI, KPIs, spend, into a structured leadership transformation brief. Covers current state assessment, future state roadmap, future RACI, technology recommendations, a four category commercial model, and a collision-safe impact vs effort chart.

Benchmark handling

The skill runs three ways. Your own benchmark register, if you provide one. A small default verified register bundled with the skill. Client baseline data with the gap stated plainly, if no verified benchmark exists for a claim. It does not invent numbers and does not block when a benchmark is missing.

What is validated in this release

Four category commercial model, never blended into one figure.
RACI governance gaps flagged, never invented owners.
Evidence confidence tagged on every KPI, RACI row, and technology entry.
Chart labels switch to a numbered legend automatically past 8 items, on label collision, or on long labels.
Full public packaging sweep passed. No legacy internal naming, no client names, no internal platform references anywhere in this repo.
